# 쇼핑몰 연동

## 프로젝트 개요
프로젝트 소개: 본 프로젝트는 개별적으로 구현되었던 '상품 목록', '장바구니', '결제' 기능을 하나의 웹 애플리케이션으로 통합하고, 신규 요구사항인 '상품 상세 페이지'를 구현하여 사용자가 끊김 없는 쇼핑 경험을 할 수 있도록 하는 것을 목표로 합니다.

참여 인원 : 정민지

## 개발 환경
Framework & Library: React 18.2.0, React Router Dom (v6), Firebase (Auth/Firestore)

Language: JavaScript (ES6+), JSX

State Management: React Context API (CartContext, PaymentContext)

Styling: Tailwind CSS

Build Tool: Create React App

## 설치 및 실행 방법
```
git clone <저장소_URL>
cd <프로젝트_폴더명>
```
패키지 설치: 필요한 의존성 패키지를 설치합니다.
```

```

실행: 개발 서버를 구동합니다.


## 프로젝트 구조
```
src/
├── components/       # 공통 UI 컴포넌트 (Header, Layout 등)
├── pages/            # 라우팅 단위 페이지
│   ├── ProductListPage.js  # 상품 목록
│   ├── ProductDetailPage.js # (신규) 상품 상세
│   └── CartPage.js         # 장바구니
├── contexts/         # 전역 상태 관리 (CartContext)
├── App.js            # React Router 설정 및 Provider 래핑
└── index.js          # 진입점
```

## 주요 함수와 기능
- App.js (라우팅 설정):
BrowserRouter와 Routes를 사용하여 /, /product/:id, /cart 경로를 설정했습니다.
모든 페이지에서 장바구니 상태를 공유하기 위해 최상단을 CartProvider로 감쌌습니다.

- useNavigate & useParams (페이지 이동 및 파라미터):
상품 목록에서 아이템 클릭 시 Maps('/product/${id}')를 통해 이동합니다.
상세 페이지에서는 useParams() 훅을 사용하여 URL의 상품 ID를 가져오고, 해당 상품의 구체적인 정보를 렌더링합니다.

- 데이터 동기화 (Context API):
ProductDetailPage에서 '장바구니 담기' 버튼 클릭 시 addToCart 함수가 호출되며 전역 상태가 업데이트됩니다.
이때 Header 컴포넌트는 페이지 이동 여부와 상관없이 갱신된 cartItems.length를 즉시 반영하여 보여줍니다.
