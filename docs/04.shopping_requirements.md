# 쇼핑몰 연동 요구사항 분석 및 기능 목록

## 📌 기본 정보
### 프로젝트명: 
패션 쇼핑몰 연동 

### 사용하게 될 기술: 
React, React Router (페이지 이동), Context API (데이터 공유)

### 문서 작성자: 
2371262

## 📝 고객 요구사항 정리
- 자연스러운 페이지 흐름: 현재 페이지별로 분리된 상품 목록, 장바구니, 결제 페이지를 사용자가 자연스럽게 이동할 수 있도록 연동해야 합니다. 
- 상세 페이지 신규 제작: 사용자가 상품 목록에서 상품을 클릭했을 때 볼 수 있는 '상세 페이지'를 새롭게 제작해야 합니다. 
- 데이터 일관성 유지 (중요): 페이지가 이동하더라도 데이터는 일관되게 유지되어야 합니다. 
(고객사 예시) 상품 목록에서 장바구니에 물건을 담으면, 이 정보가 메인 화면(헤더 등)에서도 즉시 반영되어 개수가 똑같이 보여야 합니다. 
- 테스트 URL 제공: 작업 중간 결과물을 확인할 수 있는 테스트 URL을 제공해야 합니다. 
- 디자인 시안: 상세 페이지 등을 포함한 디자인 시안은 이메일로 전달될 예정입니다. 

## 📋 기능 목록

1. 페이지 라우팅(Routing) 설정
React Router 라이브러리를 사용하여 '상품 목록', '상세 페이지', '장바구니', '결제 페이지' 간 이동 경로를 설정합니다.
상품 목록에서 특정 상품 클릭 시, 해당 상품의 고유 ID를 가진 '상세 페이지'로 이동하는 기능을 구현합니다. 

2. 상세 페이지 신규 개발
이메일로 전달받을 Figma 시안을 기반으로 '상세 페이지' UI를 개발합니다. 
(기본 기능) 상품 이미지, 이름, 가격, 상세 설명, 수량 선택, 장바구니 담기 버튼을 구현합니다.
(시안 확인) 이메일 시안에 명시된 '관련 상품(같은 브랜드) 추천' 기능을 상세 페이지 하단에 구현합니다.

3. 전역 상태 관리 (데이터 일관성)
React Context API를 사용하여 장바구니 데이터를 '전역 상태'로 관리합니다.
어떤 페이지(상품 목록, 상세 페이지 등)에서든 장바구니에 상품을 추가하면, 헤더의 장바구니 아이콘 숫자가 즉시 업데이트되도록 연동합니다. 

4. FAST URL 배포

개발된 결과물을 Vercel 또는 Netlify에 배포하여 고객사가 즉시 리뷰할 수 있는 테스트 URL을 제공합니다.
